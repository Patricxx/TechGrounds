Conflicterende Regels

    CSS staat voor Cascading Style Sheets, en dat eerste woord cascading is ongelooflijk belangrijk 
    om te begrijpen - de manier waarop de cascade zich gedraagt, is de sleutel tot het begrijpen van CSS.

    Op een gegeven moment werk je aan een project en zul je merken dat de CSS waarvan je dacht dat die 
    op een element zou moeten worden toegepast, niet werkt. Meestal is het probleem dat u twee regels 
    heeft gemaakt die mogelijk van toepassing zijn op hetzelfde element. De cascade, en het nauw verwante 
    concept van specificiteit, zijn mechanismen die bepalen welke regel van toepassing is wanneer er een 
    dergelijk conflict is. Welke regel het opmaken van je element is, is misschien niet degene die je 
    verwacht, dus je moet begrijpen hoe deze mechanismen werken.
    
    Ook belangrijk hier is het concept van overerving, wat betekent dat sommige CSS-eigenschappen standaard 
    waarden overnemen die zijn ingesteld op het bovenliggende element van het huidige element, en andere niet. 
    Dit kan ook gedrag veroorzaken dat u misschien niet verwacht.
    
    Laten we beginnen met een snelle blik op de belangrijkste dingen waarmee we te maken hebben, daarna 
    zullen we ze om beurten bekijken en zien hoe ze met elkaar en uw CSS omgaan. Dit kan een reeks lastige 
    concepten lijken om te begrijpen. Naarmate u echter meer oefent met het schrijven van CSS, wordt de 
    manier waarop het werkt voor u duidelijker.


    
Cascade

    Stylsheet Cascade - op een heel eenvoudig niveau betekent dit dat de volgorde van CSS-regels ertoe doet; 
    wanneer twee regels van toepassing zijn die dezelfde specificiteit hebben, wordt degene die als laatste in de
    CSS komt, degene die zal worden gebruikt.

    In het onderstaande voorbeeld hebben we twee regels die van toepassing kunnen zijn op de h1. De h1 wordt 
    uiteindelijk blauw gekleurd - deze regels hebben een identieke selector en hebben daarom dezelfde specificiteit, 
    dus de laatste in de bronvolgorde wint.

    h1 { 
        color: red; 
    }
    h1 { 
        color: blue; 
    }

    < h1 >This is my heading.< /h1 >


    
Specificity

    Specificiteit is hoe de browser beslist welke regel van toepassing is als meerdere regels verschillende selectors 
    hebben, maar nog steeds van toepassing kunnen zijn op hetzelfde element. Het is in feite een maatstaf voor hoe 
    specifiek de selectie van een selector zal zijn:
    
    Een elementenselector is minder specifiek - het selecteert alle elementen van dat type die op een pagina verschijnen 
    - en krijgt dus een lagere score.
    Een klassenkiezer is specifieker - hij selecteert alleen de elementen op een pagina die een specifieke 
    klasse-attribuutwaarde hebben - en krijgt dus een hogere score.
    Voorbeeld tijd! Hieronder hebben we weer twee regels die van toepassing kunnen zijn op de h1. De onderstaande h1 
    wordt uiteindelijk rood gekleurd - de klassenelector geeft zijn regel een hogere specificiteit, en dus zal deze 
    worden toegepast, ook al verschijnt de regel met de elementenselector verder naar beneden in de bronvolgorde. 

    .main-heading { 
        color: red; 
    }
            
    h1 { 
        color: blue; 
    }

    < h1 class="main-heading">This is my heading.< /h1 >

    body {
        color: blue;
    }
    
    span {
        color: black;
    }
